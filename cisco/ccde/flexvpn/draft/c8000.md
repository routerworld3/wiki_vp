Cisco 8000 Virtual Router (EC2)

Acts as the enterprise routing edge within AWS. It handles advanced Layer 3 routing (BGP, OSPF), SD-WAN integration, IPsec VPN termination, and firewalling between the tactical Wave Relay network and the broader DoD or enterprise network.

Cloud Relay Virtual Hub Router

A virtualized Wave Relay node on EC2 that bridges the proprietary MANET routing protocols into standard IP traffic. It connects directly to the Cisco router via AWS Virtual Private Cloud (VPC) subnets and Elastic Network Interfaces (ENIs).

Operational Advantages of this Integration

Advanced Traffic Engineering: The Cisco 8000v can intelligently route traffic coming from the tactical edge to different AWS availability zones, external data centers, or other cloud providers using standard enterprise protocols.
Enhanced Security Boundary: Placing the Cisco router in front of or alongside the Cloud Relay hub allows for stringent Access Control Lists (ACLs), deep packet inspection, and encrypted tunnels (IPsec/MACsec) before tactical data touches the broader enterprise network.
SD-WAN Capabilities: Integrates the tactical MANET into a broader SD-WAN fabric, allowing commanders to view and manage the tactical network as just another branch within the enterprise infrastructure.

BLUF: In this architecture, the Persistent Systems Cloud Relay Hub and the Cisco 8000 Virtual Router (8000v) operate in a complementary, gateway-to-gateway relationship within the AWS Virtual Private Cloud (VPC). The Cloud Relay acts as the tactical translator, while the Cisco router acts as the enterprise boundary.

They serve two distinctly different network domains (Tactical vs. Enterprise) and hand off traffic to each other using standard IP networking via AWS Elastic Network Interfaces (ENIs) and VPC route tables.

How They Interact (The Traffic Flow)

Because the Cisco 8000v cannot "speak" the proprietary Wave Relay protocol, and the Cloud Relay Hub is not designed to be a full-scale enterprise border gateway, they are chained together.

1. The Handoff (VPC Integration):
Within the AWS VPC, both routers are deployed as distinct EC2 instances. They are connected via a shared AWS subnet.

The Cloud Relay router points to the Cisco 8000v as its "default gateway" for any traffic destined outside the tactical network.
The Cisco 8000v uses static routing or dynamic routing (if supported by the hub's configuration) to point traffic destined for the tactical MANET subnets toward the Cloud Relay's ENI.
2. Inbound Data Flow (Tactical to Enterprise):

A user on the ground with an MPU5 radio transmits a video feed.
The data travels over SATCOM/Cellular and hits the Cloud Relay Virtual Hub.
The Cloud Relay Hub unwraps the Wave Relay routing protocol, turning it into a standard IP packet.
The Cloud Relay Hub forwards this IP packet to the Cisco 8000v.
The Cisco 8000v inspects the packet, wraps it in an IPsec tunnel, and routes it via BGP to a DoD datacenter.
3. Outbound Data Flow (Enterprise to Tactical):

A commander at a headquarters sends a Common Operating Picture (COP) update to the field.
The traffic enters the AWS VPC and hits the Cisco 8000v.
The Cisco router processes the security rules, determines the destination is on the tactical network, and forwards the traffic to the Cloud Relay Virtual Hub.
The Cloud Relay Hub ingests the standard IP traffic, applies Wave Relay routing algorithms, and multicasts it down to the appropriate MPU5 radios in the field.
Why Use Both?

Using both allows network engineers to maintain a strict boundary between the tactical radio environment and the enterprise backbone. It allows the tactical units to use the specialized equipment they need (Wave Relay) without forcing the enterprise IT department to manage proprietary routing protocols on their core network.
